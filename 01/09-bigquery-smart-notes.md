# 🧠 Q11: Choosing a Serverless SQL Analytics Tool for Large-Scale Data

## 📌 Question
Cymbal Retail collects large amounts of data that is useful for improving business operations.  
The company wants to store and analyze this data in a **serverless** and **cost-effective** manner using Google Cloud.  
The analysts also need to use **SQL** to write queries.  

👉 What tool should you use?

A. Data Fusion  
B. BigQuery  
C. Spanner  
D. Memorystore  

---

## ✅ Correct Answer
👉 **B. BigQuery**

---

## 🧠 Core Concept
👉 **Serverless + SQL + Large-scale analytics = BigQuery**

---

## ⚙️ Why BigQuery is Correct

BigQuery is:
- ✔️ Fully serverless (no infrastructure management)
- ✔️ Designed for large-scale data (TBs–PBs)
- ✔️ Supports standard SQL
- ✔️ Cost-effective (pay per query)

👉 Analysts can directly run SQL queries on massive datasets  
👉 No need to manage servers or clusters  

💥 Perfect match for all requirements

---

## ❌ Why Other Options Are Wrong

### A. Data Fusion
- ❌ Used for building pipelines (GUI)
- ❌ Not for querying/analyzing data  

---

### C. Spanner
- ❌ Transactional database (OLTP)
- ❌ Not optimized for analytics  

---

### D. Memorystore
- ❌ In-memory cache (Redis/Memcached)
- ❌ Not for data storage + analytics
